## 概述

`paper-search-mcp` 集成了 **21 个学术数据源**（全部已实现），覆盖计算机科学、生物医学、物理学、社会科学等多个领域。数据源分为三种能力层级：

| 能力层级 | 说明 | 数量 |
|----------|------|------|
| **可搜索 + 可下载 PDF + 可读取** | 完整链路支持 | 8 个 |
| **可搜索 + 条件性 PDF** | 取决于具体记录是否有 OA PDF | 8 个 |
| **仅可搜索（元数据）** | 提供发现和 DOI 路由，不托管 PDF | 5 个 |

另外还有 **2 个骨架来源**（IEEE、ACM）需要 API Key 才能使用，以及 **Unpaywall**（OA URL 解析器）和 **Sci-Hub**（用户主动启用的兜底下载器）。

## 按能力分类

### 可直接下载 PDF 的数据源（8 个）

这些数据源提供完整的"搜索 → 下载 → 读取"链路，是最可靠的选择：

| 数据源 | 适用领域 | 可靠性 | 说明 |
|--------|----------|--------|------|
| **arxiv** | CS / 通用 | ⭐⭐⭐⭐⭐ | **唯一对 CS 领域 PDF 下载稳定可靠的数据源** |
| pmc | 生物医学 OA | ⭐⭐⭐⭐ | 仅生物医学开放获取论文 |
| biorxiv | 生物学预印本 | ⭐⭐⭐⭐ | 仅生物学领域 |
| medrxiv | 医学预印本 | ⭐⭐⭐⭐ | 仅医学领域 |
| europepmc | 生物医学 OA | ⭐⭐⭐ | 取决于是否有 OA 版本 |
| iacr | 密码学预印本 | ⭐⭐⭐ | 仅密码学，直接下载可能受限 |
| semantic | 通用 | ⭐⭐⭐ | 仅在 openAccessPdf 可用时 |
| core | 通用仓储 | ⭐⭐ | 依赖具体记录，推荐配置 API Key |

### 仅元数据，无法下载 PDF（5 个）

这些数据源用于论文发现和 DOI/标题信息补充，需要通过 fallback 链下载 PDF：

| 数据源 | 说明 |
|--------|------|
| **crossref** | DOI 元数据主干，不托管 PDF。返回的论文条目可路由到其他来源下载 |
| **openalex** | 元数据和 OA 链接发现。常指回 arXiv PDF |
| **dblp** | 计算机科学文献元数据，高质量 CS 参考文献，用于 DOI/标题补充 |
| **google_scholar** | 发现引擎，不直接提供 PDF。反爬严格 |
| **pubmed** | 生物医学元数据，可通过 DOI/PMC 路由到下载来源 |
| **chemrxiv** | 化学预印本，通过 Crossref 过滤实现，仅元数据 |
| **openaire** | OA 发现链接，直接工具仅提供元数据 |
| **unpaywall** | DOI → OA URL 解析器，不托管 PDF。需配置 Email |

### 下载不稳定（record_dependent）（8 个）

| 数据源 | CS 覆盖率 | 说明 |
|--------|-----------|------|
| semantic | 中等 | 取决于 Semantic Scholar 是否索引到 openAccessPdf |
| core | 低 | 仓储记录依赖具体 PDF 可用性 |
| citeseerx | 低 | 旧版 CS 来源，当前访问路径容易超时 |
| doaj | 低 | OA 期刊记录，PDF 需通过期刊网站获取 |
| base | 低 | OAI-PMH，可能需要注册 IP |
| zenodo | 低 | 开放仓储文件，CS 论文较少 |
| hal | 低 | 法国 OA 文档，CS 论文较少 |
| ssrn | 极低 | 登录/反爬限制，仅公开 PDF 可下载 |

## 搜索配置文件 (Search Profiles)

通过 `PAPER_SEARCH_MCP_SEARCH_PROFILE` 环境变量或工具参数 `sources` 选择预设的数据源组合：

| Profile 名称 | 别名 | 数据源 | 适用场景 |
|-------------|------|--------|----------|
| **`fast`** | `default` | arxiv, semantic, openalex, crossref, pubmed, pmc, europepmc | 通用快速检索，覆盖多学科（7 个源） |
| **`pdf-cs`** | `cs-pdf`, `pdf_cs`, `cs_pdf` | arxiv, openalex, crossref, dblp | 计算机科学 PDF 优先检索（4 个源） |
| **`agent-skill-fast`** | `agent_skill_fast` | arxiv, openalex, crossref | 最小集，适合 Agent 快速决策（3 个源） |
| **`agent-skill-broad`** | `agent_skill_broad` | arxiv, openalex, crossref, semantic, google_scholar | Agent 广泛检索（5 个源） |
| **`deep`** | `all` | 全部 21 个数据源 | 全面检索，耗时较长 |

### 配置方式

**方式一：环境变量（全局默认）**

```dotenv
# 在 .env 中设置
PAPER_SEARCH_MCP_SEARCH_PROFILE=pdf-cs
```

**方式二：工具参数（单次调用）**

```json
{
  "tool": "search_papers",
  "arguments": {
    "query": "transformer model compression",
    "sources": "deep"
  }
}
```

**方式三：自定义来源组合**

```json
{
  "tool": "search_papers",
  "arguments": {
    "query": "graph neural network",
    "sources": "arxiv,semantic,dblp"
  }
}
```

### 禁用特定数据源

```dotenv
# 禁用不需要的数据源（逗号分隔）
PAPER_SEARCH_MCP_DISABLED_SOURCES=pubmed,biorxiv,medrxiv,google_scholar,ssrn
```

## 各数据源详细说明

### 1. arXiv (`arxiv`)

- **API 端点**：`http://export.arxiv.org/api/query`
- **能力**：搜索 ✅ | 下载 ✅ | 读取 ✅
- **认证**：无需 API Key
- **超时配置**：`ARXIV_TIMEOUT_SECONDS`（默认 8 秒）、`ARXIV_MAX_ATTEMPTS`（默认 2 次）
- **说明**：最重要的 CS 数据源，提供稳定的 PDF 下载和论文文本读取。支持 `download_arxiv`、`read_arxiv_paper` 等专用工具。
- **适用场景**：所有计算机科学领域的论文检索

### 2. Semantic Scholar (`semantic`)

- **API 端点**：`https://api.semanticscholar.org/graph/v1/paper/search`
- **能力**：搜索 ✅ | 下载 ⚠️（仅 openAccessPdf） | 读取 ⚠️
- **认证**：可选 `SEMANTIC_SCHOLAR_API_KEY`（提升速率限额）
- **配置**：`PAPER_SEARCH_MCP_SEMANTIC_SCHOLAR_API_KEY`
- **说明**：通用学术搜索引擎。未认证可用但有速率限制；下载仅在返回 `openAccessPdf` 字段时可用。提供引用计数、高引论文等信息。
- **注意**：即使不配置 Key，搜索功能仍可用，但频率受限

### 3. Crossref (`crossref`)

- **API 端点**：`https://api.crossref.org/works`
- **能力**：搜索 ✅ | 下载 ❌ | 读取 ❌
- **认证**：无需 API Key
- **说明**：DOI 元数据主干，提供完整的论文元数据（标题、作者、期刊、DOI、参考文献等）。虽然不能直接下载 PDF，但返回的 DOI 可路由到其他来源。
- **适用场景**：DOI 查找、元数据补充

### 4. OpenAlex (`openalex`)

- **API 端点**：`https://api.openalex.org/works`
- **能力**：搜索 ✅ | 下载 ❌ | 读取 ❌
- **认证**：无需 API Key
- **说明**：开放学术数据目录，提供丰富的元数据和 OA 链接。常指向 arXiv、PMC 等来源的 PDF。
- **适用场景**：OA 论文发现、元数据检索

### 5. dblp (`dblp`)

- **API 端点**：`https://dblp.org/search/publ/api`（XML）+ HTML fallback
- **能力**：搜索 ✅ | 下载 ❌ | 读取 ❌
- **认证**：无需 API Key
- **说明**：计算机科学文献的高质量元数据来源，提供精确的作者、会议/期刊信息。不托管 PDF，但提供 DOI 用于路由。
- **适用场景**：CS 特定论文发现、作者/会议信息查询

### 6. PubMed (`pubmed`)

- **API 端点**：`https://eutils.ncbi.nlm.nih.gov/entrez/eutils/esearch.fcgi` + `efetch.fcgi`
- **能力**：搜索 ✅ | 下载 ❌ | 读取 ❌
- **认证**：无需 API Key
- **说明**：生物医学文献的权威元数据源。不直接提供 PDF，但可通过 DOI 路由到 PMC 或其他 OA 来源。
- **适用场景**：生物医学领域的论文发现

### 7. PubMed Central - PMC (`pmc`)

- **API 端点**：`https://eutils.ncbi.nlm.nih.gov/entrez/eutils/esearch.fcgi` + `esummary.fcgi`
- **能力**：搜索 ✅ | 下载 ✅（OA PDF） | 读取 ✅
- **认证**：无需 API Key
- **说明**：PubMed Central 的 OA 子集，可下载开放获取的全文 PDF。
- **适用场景**：生物医学 OA 论文的获取

### 8. Europe PMC (`europepmc`)

- **API 端点**：`https://www.ebi.ac.uk/europepmc/webservices/rest/search`
- **能力**：搜索 ✅ | 下载 ⚠️（OA PDF） | 读取 ⚠️
- **认证**：无需 API Key
- **说明**：欧洲版 PMC，覆盖面更广。提供 OA PDF 下载和论文文本读取。
- **适用场景**：生物医学领域的欧洲补充来源

### 9. bioRxiv (`biorxiv`)

- **API 端点**：`https://api.biorxiv.org/details/biorxiv/`
- **能力**：搜索 ✅ | 下载 ✅ | 读取 ✅
- **认证**：无需 API Key
- **说明**：生物学预印本服务器，支持按分类筛选、PDF 下载和文本读取。
- **适用场景**：生物学最新预印本

### 10. medRxiv (`medrxiv`)

- **API 端点**：`https://api.biorxiv.org/details/medrxiv/`
- **能力**：搜索 ✅ | 下载 ✅ | 读取 ✅
- **认证**：无需 API Key
- **说明**：医学预印本服务器，与 bioRxiv 共享 API 架构。
- **适用场景**：医学最新预印本

### 11. ChemRxiv (`chemrxiv`)

- **API 端点**：使用 Crossref API（过滤至 ChemRxiv 出版商）
- **能力**：搜索 ✅ | 下载 ❌ | 读取 ❌
- **认证**：无需 API Key
- **说明**：化学预印本，通过扩展 Crossref 搜索实现。仅提供元数据。
- **适用场景**：化学预印本发现

### 12. Google Scholar (`google_scholar`)

- **API 端点**：`https://scholar.google.com/scholar`（HTML 抓取）
- **能力**：搜索 ✅ | 下载 ❌ | 读取 ❌
- **认证**：可选 `GOOGLE_SCHOLAR_PROXY_URL`（代理 URL）
- **配置**：`PAPER_SEARCH_MCP_GOOGLE_SCHOLAR_PROXY_URL`
- **说明**：最广泛的学术搜索引擎，但无官方 API，通过 HTML 页面抓取实现。容易触发反爬检测。仅用于论文发现，不提供直接 PDF。
- **注意**：默认在 `pdf-cs` profile 中被禁用。速度慢且不稳定

### 13. IACR ePrint (`iacr`)

- **API 端点**：`https://eprint.iacr.org/search`（HTML 抓取）
- **能力**：搜索 ✅ | 下载 ✅ | 读取 ✅
- **认证**：无需 API Key
- **说明**：国际密码学研究协会的预印本存档。提供密码学领域的 PDF 下载和读取。
- **适用场景**：密码学领域的论文检索

### 14. CORE (`core`)

- **API 端点**：`https://api.core.ac.uk/v3/search/works`
- **能力**：搜索 ✅ | 下载 ⚠️（记录依赖） | 读取 ⚠️
- **认证**：推荐 `CORE_API_KEY`
- **配置**：`PAPER_SEARCH_MCP_CORE_API_KEY`
- **说明**：全球最大的 OA 论文仓储聚合器。未配置 Key 时功能受限，推荐配置。
- **适用场景**：OA 论文的广泛检索

### 15. OpenAIRE (`openaire`)

- **API 端点**：`https://api.openaire.eu/search/researchProducts`
- **能力**：搜索 ✅ | 下载 ❌ | 读取 ❌
- **认证**：可选 `OPENAIRE_API_KEY`
- **配置**：`PAPER_SEARCH_MCP_OPENAIRE_API_KEY`
- **说明**：欧洲开放科学基础设施，提供研究产品的 OA 发现链接。
- **适用场景**：欧洲研究项目的论文发现

### 16. CiteSeerX (`citeseerx`)

- **API 端点**：`https://citeseerx.ist.psu.edu/api/search`
- **能力**：搜索 ✅ | 下载 ⚠️（记录依赖） | 读取 ⚠️
- **认证**：可选 `CITESEERX_API_KEY`
- **配置**：`PAPER_SEARCH_MCP_CITESEERX_API_KEY`
- **说明**：老牌 CS 文献搜索引擎。当前访问路径容易超时，CS 覆盖率低。
- **注意**：速度和稳定性较差，不建议作为主要来源

### 17. DOAJ (`doaj`)

- **API 端点**：`https://doaj.org/api/search/articles/`
- **能力**：搜索 ✅ | 下载 ⚠️（URL 依赖） | 读取 ⚠️
- **认证**：可选 `DOAJ_API_KEY`
- **配置**：`PAPER_SEARCH_MCP_DOAJ_API_KEY`
- **说明**：开放获取期刊目录，提供 OA 期刊文章记录。PDF 需通过期刊网站获取。
- **适用场景**：OA 期刊论文发现

### 18. BASE (`base`)

- **API 端点**：`https://api.base-search.net/cgi-bin/BaseHttpSearchInterface.fcgi`（OAI-PMH）
- **能力**：搜索 ⚠️（机构依赖） | 下载 ⚠️（记录依赖） | 读取 ⚠️
- **认证**：无需 API Key（可能需要注册 IP）

## 下载 Fallback 链

当通过 `download_with_fallback` 或 `download_selected_papers` 下载论文时，系统按以下优先级尝试：

```
1. 来源原生下载（如 arxiv.download_pdf）
   ↓ 失败
2. OA 仓储竞速下载（CORE、OpenAIRE、Europe PMC、PMC 并发尝试）
   ↓ 失败
3. Unpaywall DOI OA URL 解析（需配置 UNPAYWALL_EMAIL）
   ↓ 失败
4. paper_fetch 兜底（默认禁用，需 PAPER_SEARCH_MCP_PAPER_FETCH_PDF_FALLBACK=true）
   ↓ 失败
5. Sci-Hub 兜底（默认禁用，需用户明确传 use_scihub=true）
```

下载过程中，三个主要方法（primary、repositories、unpaywall）会并发竞速，谁先成功就使用谁的结果，其余取消。

```json
{
  "tool": "diagnose_paper_sources",
  "arguments": {}
}

- 是否被 `DISABLED_SOURCES` 禁用
- API Key 配置状态
- 可靠性评分
- 推荐操作（是否需要配置 Key、仅用于元数据等）

{
  "tool": "list_sources",
  "arguments": {}
}
```

返回所有支持的数据源列表及其能力概况。
